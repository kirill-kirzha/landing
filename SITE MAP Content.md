### *Flows*

**Hero headline:**
Business logic that runs the same way every time.

**Hero subline:**
A production-grade execution engine for the processes your organization cannot afford to get wrong.

*Video*

**The problem:**
Every enterprise has critical processes that run on tribal knowledge, manual steps and one-off automation scripts that break whenever something changes. When AI gets added, it makes the output smarter but the process no more reliable. Flows solves that. It is the place where business logic becomes repeatable execution.

**What it Offers:**

> **Automate a chain of tasks**
>
> A Flow takes a typed input payload and turns it into a reliable output by executing a sequence of nodes — observable, versioned and safe to run in production.

> **Trigger**
> Manual, scheduled (cron) or webhook from any external system. Every run starts with a defined, typed input.

> **Canvas Graph**
> Directed graph with branching logic — built visually, executed deterministically.

> **Nodes — the building blocks:**
> - Data nodes: SQL queries, database lookups
> - Document nodes: PDF extraction, file parsing
> - Model nodes: LLM with explicit constraints and prompts
> - Tool nodes: calculations, formatting, transformations
> - Write-back nodes: update Salesforce, SAP, Oracle or any connected system
> - Agent Node: embed an AI Employee as an adaptive step inside a controlled process

> **Structured Output**
> Every Flow ends with a schema-validated JSON output — making Flows a production-grade backend that powers dashboards, applications and downstream automations.

> **Run Trace**
> Every run logged with full step-by-step trace, input/output snapshots and timing. Real-time observability, not retroactive debugging.

> **Versioning**
> Save, promote and rollback. Test runs validate before production. No change goes live without confirmation.

> **Authoring:**
> Build manually with explicit prompts, SQL queries and parameters — for power users who need full control.
> Or use Assisted Authoring: describe what you want, and the system generates the node configuration. Either way, what ships is controlled enough for production.

## FAQ

**What is included in the base platform vs add-ons?**
The base platform (Personal or Business) gives you Aleria OS — chat, sources, reports and core agentic capabilities. Add-ons extend this with specialized infrastructure: Flows for workflow automation, Datalake for live database connections, Agentic Workforce for background AI Employees, and ETL for large-scale data pipelines.

**Can I deploy fully on-premise with no cloud dependency?**
Yes. Aleria is designed for on-premise and air-gapped deployment. Your data never leaves your infrastructure. Zero external API calls are made during operation.

**What does air-gapped delivery mean in practice?**
Aleria is delivered as an encrypted VM on your hardware — Mac Studio, NVIDIA DGX or your own servers. It operates with zero internet connectivity. The entire platform runs locally. This has been deployed and is running in production at multiple government and financial institutions.

**Is there a minimum seat requirement?**
Personal plans start at 1 seat. Business plans are per seat with no minimum. Enterprise and Custom Solution deployments are scoped per organization.

**What AI models does Aleria use?**
Aleria is model-agnostic. We support and integrate the leading LLMs and allow organizations to bring their own models. In air-gapped deployments, only locally hosted models are used — no external model API calls.

**Can I bring my own model?**
Yes. Aleria's architecture is built to be model-agnostic. Enterprise deployments can run fully on locally hosted models with no dependency on any external AI provider.

**How long does onboarding take?**
Tier 1 standard use cases can be live in hours. Tier 2 configured deployments take days to weeks. Enterprise and sovereign deployments are scoped individually — Aleria provides a dedicated implementation team for all Custom Solution contracts.

**What exactly is an AI Employee?**
An AI Employee is a role-based autonomous agent that operates in a controlled execution environment. Unlike a chatbot, it doesn't just answer questions — it opens files, runs code, queries databases, executes multi-step tasks and delivers finished outputs. It works in background mode, handles unlimited files and can be customized with skills and templates specific to your organization's processes.
